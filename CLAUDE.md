# HTL プロジェクト

## 概要

LPIC-1 試験対策用の VitePress 静的サイト。
研修資料（Markdown）をWebサイトとして公開し、学習者が事前学習に使う。

- **ローカル開発**: `npm run dev` → `http://localhost:5173/HTL/`
- **本番URL**: https://esx11954.github.io/HTL/
- **リポジトリ**: https://github.com/esx11954/HTL

## 技術スタック

- **VitePress** 1.x（静的サイトジェネレーター）
- **markdown-it-task-lists**（チェックボックスのレンダリング）
- **GitHub Actions + GitHub Pages**（自動デプロイ）

## 主な実装ポイント

### GitHub Pages へのデプロイ
`config.mts` に `base: '/HTL/'` が必要。これがないと CSS・JS のパスがずれて表示が崩れる。

### インタラクティブチェックボックス
- `markdown-it-task-lists` プラグインで `- [ ]` を `<input type="checkbox">` にレンダリング
- `Layout.vue` でページ読み込み時にチェックボックスの `disabled` を解除
- チェック状態を `localStorage` に保存（キー: `checkbox_{path}_{index}`）
- ページ遷移時も `watch(() => route.path, ...)` で再初期化

### 画像の扱い
元資料の画像参照は `images/XX_name.png` 形式。
`docs/lpic101/images/` に配置すれば Markdown からそのまま参照できる。

## 想定問題演習ページ（HTML）の扱い

- 元資料が HTML ファイルで納品される場合は `docs/public/chapterN.html` に配置する
- VitePress の SPA ルーターと競合するため、サイドバーは `target: '_blank'` で新タブ開きにする
- サイドバーリンクのスタイルは `custom.css` でティール色・下線・背景ハイライトを適用済み（`a[target="_blank"]` セレクタ）
- 元ファイルの差し替えは `cp` で上書きしてコミット・push するだけでよい
- 現在の配置: `docs/public/chapter6.html`、`docs/public/chapter13.html`

### パスワード保護（未実装・方針のみ）

問題ページ（chapter6.html・chapter13.html）にはパスワードを設ける予定。

**方針：ゲートページ方式**
1. `docs/public/practice-gate.html` を作成（パスワード入力フォーム）
2. 正解時に `sessionStorage` へトークンを保存し、対象ページへリダイレクト
3. 各問題 HTML はロード時にトークル確認 → なければゲートページへリダイレクト
4. パスワードは SHA-256 ハッシュ化して JS に埋め込む（平文は保存しない）

**注意：** 静的サイトのため完全なセキュリティは担保できない。研修参加者以外が容易にアクセスできない程度の抑止力として割り切る。

## コンテンツ共通ルール

- **ページタイトル（`#` h1）はサイドバーの `text` と完全に一致させる**
- **目次セクション（`## 目次` ～リスト）は HTML コメント `<!-- ... -->` でコメントアウトする**

## コンテンツ追加の手順

新しい資料（例: 第4章）を追加する場合：

1. 元資料（`HTLfiles/HTLFilesN/`）の Markdown と画像を確認する
2. 元 Markdown を `cp` で `docs/lpic101/chapterN.md` にコピーする（**Markdown をゼロから書かない。このプロジェクトで Markdown を新規作成することは絶対にない**）
3. h1 タイトルを `# 第N章：〜` 形式に修正する（サイドバーの `text` と完全一致させる）
4. 目次セクション（`## 目次` ～リスト末尾）を HTML コメント `<!-- ... -->` でコメントアウトする
5. 画像を `docs/lpic101/images/` にコピーする
6. `docs/.vitepress/config.mts` のサイドバーにエントリを追加する
7. コミット → push → GitHub Actions が自動デプロイ
